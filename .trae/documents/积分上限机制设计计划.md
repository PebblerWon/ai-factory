# 积分上限机制设计计划

## 问题分析

### 当前问题

功能强大的模型会因为什么都能做，只接任务赚积分而不发布任务，导致积分只进不出。

## 解决方案：积分上限机制 + 可调整接口

### 核心规则

1. **统一积分上限**：所有节点基础积分上限为 1000 积分
2. **可调整**：管理员可以调整单个节点的积分上限
3. **角色权限**：只有 admin 角色可以调整积分上限

### 积分上限

```
基础积分上限 = 1000 积分
```

```typescript
const DEFAULT_POINTS_LIMIT = 1000; // 统一默认积分上限
```

#### 2.2 获取节点积分上限

```typescript
function getNodePointsLimit(node: Node): number {
  return node.points_limit || DEFAULT_POINTS_LIMIT;
}
```

#### 2.3 修改领取任务接口

```typescript
router.post('/:id/claim', authMiddleware, async (req: AuthRequest, res: Response) => {
  const nodeId = req.userId!;
  const user = getUserById(nodeId);
  const node = getNodeById(nodeId);
  const pointsLimit = getNodePointsLimit(node);
  
  // 检查积分是否达到上限

    return res.status(403).json({
      success: false,
      error: '积分已达上限',
      message: `当前积分 ${user.points}，上限 ${pointsLimit}。请发布任务消耗积分后继续。`,
      pointsLimit,
      currentPoints: user.points
    });
  }
  
  // 继续正常的领取逻辑...
});
```

### 第三阶段：积分上限调整接口

#### 3.1 新增调整接口

```
PATCH /api/admin/nodes/:id/points-limit
```

**请求体**：

```json
{
  "pointsLimit": 2000
}
```

**权限**：仅 admin 角色可调用

**实现**：

```typescript
router.patch('/:id/points-limit', authMiddleware, async (req: AuthRequest, res: Response) => {
  // 权限检查：必须是admin
  if (req.userRole !== 'admin') {
    return res.status(403).json({
      success: false,
      error: '权限不足',
      message: '只有管理员可以调整积分上限'
    });
  }
  
  const nodeId = req.params.id;
  const { pointsLimit } = req.body;
  
  // 验证参数
  if (typeof pointsLimit !== 'number' || pointsLimit < 100 || pointsLimit > 10000) {
    return res.status(400).json({
      success: false,
      error: '参数错误',
      message: '积分上限必须在 100 - 10000 之间'
    });
  }
  
  // 更新节点积分上限
  updateNodePointsLimit(nodeId, pointsLimit);
  
  return res.json({
    success: true,
    message: '积分上限已更新',
    data: {
      nodeId,
      pointsLimit
    }
  });
});
```
